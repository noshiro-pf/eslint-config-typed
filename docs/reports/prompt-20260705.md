- (mutating method を `Array.from(xs)` （但し xs は配列型）の直後に使用している場合、 対応する no-mutating method に置き換える fixer 付き既存ルール)があるかどうか調査し、なければ追加してください（あるとしたら unicorn や typescript-eslint あたりにありそうです）。 カスタムルールを追加する場合は ts-restrictions）に追加。
    - `Array.from(xs).reverse()` -> `xs.toReversed()`
    - `Array.from(xs).sort()` -> `xs.toSorted()`
    - `Array.from(xs).fill(v)` -> `xs.map(() => v)`
    - `Array.from(xs).splice(...args)` -> `xs.toSpliced(...args)`
- 配列への非破壊要素追加／削除を行う記法を統一するルールを ts-data-forge 向け custom plugin に追加してください。置き換え対象として考えられるパターンをなるべく列挙しましたが、他にも考えられる場合は追加してください。
    - 先頭の1要素削除 -> 以下パターンを `Arr.tail(xs)` に統一
        - `xs.toSpliced(0, 1)`
        - `xs.slice(1)`
        - `xs.filter((_, i) => i >= 1)`
        - `xs.filter((_, i) => i > 0)`
        - `xs.filter((_, i) => i !== 0)`
        - `Arr.skip(xs, 1)`
    - 先頭の N 要素 削除 -> `Arr.skip(xs, N)` に統一
        - `xs.toSpliced(0, N)`
        - `xs.slice(N)`
        - `xs.filter((_, i) => i >= N)`
    - 先頭の N 要素 残し -> `Arr.take(xs, N)` に統一
        - `xs.toSpliced(N)`
        - `xs.slice(0, N)`
        - `xs.filter((_, i) => i < N)`
    - 末尾の1要素削除 -> `Arr.butLast(xs)` に統一
        - `xs.toSpliced(-1)`
        - `xs.slice(0, -1)`
        - `xs.filter((_, i) => i < xs.length - 1)`
        - `xs.filter((_, i) => i !== xs.length - 1)`
        - `Arr.skipLast(xs, 1)`
    - 末尾のN要素削除 -> `Arr.skipLast(xs, N)` に統一
        - `xs.toSpliced(-N)`
        - `xs.slice(0, -N)`
        - `xs.filter((_, i) => i < xs.length - N)`
    - 末尾の N 要素 残し -> `Arr.takeLast(xs, N)` に統一
        - `xs.toSpliced(0, -N)`
        - `xs.slice(-N)`
        - `xs.filter((_, i) => i >= xs.length - N)`
    - 先頭への1要素追加 -> `Arr.toUnshifted(xs, newItem)` に統一
        - `[newItem, ...xs]`
        - `[newItem].concat(xs)`
        - `xs.toSpliced(0, 0, newItem)`
    - 末尾への1要素追加 -> `Arr.toPushed(xs, newItem)` に統一
        - `[...xs, newItem]`
        - `xs.concat(newItem)`
        - `xs.toSpliced(xs.length, 0, newItem)`
